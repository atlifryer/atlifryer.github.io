Project 1 — Building my website

Project Description

The objective of this first project of VÉL403G was to create a website using HTML. This website will be used for putting the rest of the projects that will be done in this course and also to upload my resume. Learning how to make my own website is very useful because I want to use this website for myself after I graduate in the spring. I've spent quite some time on this website to make it look as nice as possible and I've learned a lot about how HTML, JavaScript and CSS works, though mostly I've come to understand HTML better.

Making the Website itself

I have no prior experience using HTML except for sometimes seeing some snippets here and there when browsing the internet, so I decided to use a template called Editorial that you can see for yourselves here. After looking at the examples given on Canvas of good websites I decided to download the application Brackets for some extra help with the coding of the website. I also spent some time looking up how to make some fancier parts of the website, such as the image slider on the front page and the smooth scrolling when clicking on the "About me" section. I decided against having too many pages so the only real page apart from this one is the front page. Following the instructions given by the teacher and using the websites from last year I started by downloading Brackets, making an account at Github. Then I installed Homebrew to my computer and followed the instructions given on this website to connect my website to my repository. This results in a file called README.md to appear in the folder that Brackets is working with. After this I found my template at HTML5 UP, and started to work on the website. The rest of my time went into making the page look as clean as possible for future usage.

Uploading and updating the website

Once I had finished the website there were a few snippets of code left to insert into Terminal. Firstly you should check that you are working with the right directory, and then you can insert these 4 simple lines of code:
git add .
git status
git commit -m assisted
git push
The picture below shows what my Terminal looks like when I try to update the page after changing nothing:

The git status command is unnecessary, but it's useful to confirm which changes had been made. Once this was done, the only thing left was to wait a little bit as it takes some time for the page to be fully updated, but after that, the page is ready! These 4 lines of code are also used to update the page after any changes to it.
